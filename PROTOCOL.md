# NOVAX PROTOCOL

## 1. Unidades
NOVAX utiliza unidades digitales llamadas NVX.
No existen tokens externos ni contratos de terceros.

## 2. Identidad
Cada usuario posee una identidad única representada por una wallet.
La wallet es creada localmente y no depende de correos públicos.

## 3. Wallets
- Las wallets se generan mediante claves internas
- No se pueden inventar wallets
- Si una wallet no existe, el sistema la rechaza

## 4. Registro
Todas las operaciones quedan registradas de forma pública
en un historial inmutable.

## 5. Emisión
La emisión es limitada y controlada por reglas fijas.
No se permite creación arbitraria.

## 6. Participación
Cualquier usuario puede participar desde un dispositivo móvil,
siempre que respete las reglas del protocolo.

## 7. Seguridad
- Claves privadas nunca se comparten
- No hay recuperación automática
- La responsabilidad es del usuario

## 8. Evolución
Las reglas pueden ampliarse,
pero nunca romper lo ya documentado.
